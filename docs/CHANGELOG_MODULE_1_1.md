## Also fixed along the way (not separately numbered in the review, found during this pass)

- `components/shipments/new-shipment-form.tsx`'s original supplier-added handler called a state-setter during render (`onCreated(supplier)` directly in a component body) — a real React anti-pattern that would have caused a "cannot update state during render" warning/bug. Moved into a `useEffect`.
- Extracted `CreateShipmentSchema`/`AddSupplierSchema`/`LoginSchema` out of the `"use server"` action files into plain `lib/schemas/` modules — needed to make them unit-testable at all (a `"use server"` file can't be imported directly in a Vitest environment), and a reasonable separation of concerns regardless.
